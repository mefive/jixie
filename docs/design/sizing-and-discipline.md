## 6.1 仓位管理实验室

### 为什么仓位是「系统的一半」

高赔率系统(趋势/动量)靠少数大赢覆盖多数小亏,同一套信号配不同仓位,结果可以从「翻倍」到「爆回撤」。仓位方案决定的是**每次亏损占权益的比例**和**回撤的深度分布**,与进出场信号同等重要。

### SDK:中性 sizing 原语(不预设哪种好——产品哲学)

```ts
// ATR 风险仓位(海龟式):买入股数使「1 个 ATR 的不利波动 ≈ 权益的 riskPct」
// shares = equity × riskPct / ATR(n),取整手
ctx.atrUnits(code, riskPct, atrPeriod?): number

// 波动率目标权重(横截面):按个股波动率倒数缩放,使组合各持仓风险贡献趋近
// 输入候选列表,返回 Map<code, weight>(和为 1),喂给 setHoldings
volTargetWeights(codes, ctx, lookback?): Promise<Map<string, number>>
```

- 已有:`equalWeight`(等权)、固定 unit(用户代码自己写,曾庆辉 2M/20unit 即例)。
- **Kelly 不进 SDK**:Kelly 需要胜率/赔率估计,样本误差极大,full-Kelly 对个人是危险品;放在**报告侧**(原计划随 6.2 回撤画像顺带输出「按 tradeLog 估计的 Kelly 分数与 1/2 Kelly」,附警示文案;6.2 已搁置,此项随之待定)。
- 新原语同步 SDK 三镜像(建议 4.4 单一来源先行)。

### 对比工具:骑在参数扫描(1.2)上

仓位方案本质是一种参数。实现上不建新系统:

- 策略代码把 sizing 提为参数(如 `params.riskPct` 或 `params.sizing: 'equal' | 'atr' | 'vol'`,用户代码内 switch);
- 参数扫描跑网格 → **仓位对比视图**是扫描结果的一个专用呈现:多方案净值叠画 + 最深回撤 / 水下期 / 年化波动 / Sharpe 对照表。
- 前端归在 lab 的「参数扫描」入口下,不单开页面(单用户研究,少一层导航)。

### 验收

同一唐奇安/海龟信号,等权 vs ATR 风险仓位 vs 固定 unit 三方案一次扫描出对照;ATR 方案的回撤分布应显著更受控(教科书行为)。

## 6.2 回撤画像 + 预承诺卡 💤(2026-07-06 判定伪需求,先不做)

> **搁置**:① 回测面板已有实际最大回撤 + 月度收益热力图,「跨行情段看表现」基本已覆盖;② bootstrap 重排毁掉真实收益顺序(回撤对顺序敏感),打乱后已非历史,虚构统计价值可疑——用户判定伪需求。唯一真缺的是回撤的**水下时长/恢复期**(热力图看不出),价值有限,等真要上线某个策略时再顺手加,不单列。下文原设计保留备查。

### 是什么(通俗)

回测给的是**一条**历史路径;把这条路径的日收益做 bootstrap 重采样(块状抽样,保留短期相关性),生成如 1000 条「平行历史」,统计:

- 最深回撤的分布(中位数 / 95% 分位)——「运气差一点时会有多深」;
- 最长水下期(净值创新高之间的最长间隔);
- 连亏笔数分布(按 tradeLog 配对盈亏)、单月最大亏损;
- (顺带)Kelly 估计,见 6.1。

### 为什么

**事前知道的疼是纪律,事后遭遇的疼是恐慌。** 弃系统几乎都发生在「没想到会这么疼」的时刻;把「正常会多疼」量化在前,回撤发生时才有对照系。这也是学习清单阶段 5(过拟合/稳健)的活教材:一条历史路径的 maxDD 是随机变量的一次抽样,不是承诺。

### 实现要点

- 纯统计,引擎零改动:输入 = 现有 `BacktestSummary` 的 nav + tradeLog。
- 块状 bootstrap(block bootstrap,块长 ~20 交易日)优先于逐日独立抽样——日收益有自相关,独立抽样会低估回撤。实现是几十行纯函数,进 `lib/stats.ts`,vitest 单测。
- 呈现两处:
  1. 回测结果面板加「回撤画像」折叠区(直方图 + 关键分位数);
  2. **预承诺卡**:策略「上线」(live 开关,主线五)时弹确认页——「该系统历史上:最深回撤 X%,95% 置信最深 W%;最长水下 Y 个月;每年正常连亏 Z 笔。确认上线即表示接受以上为正常波动。」文案中性陈述,不劝进不劝退。
- 计算量小(1000 × 数千点),可同步算或塞进回测 worker 收尾。

### 验收

高换手趋势策略与低换手价值策略的画像呈现出教科书差异(前者连亏长、后者水下期长);预承诺卡在开 live 时出现一次。

## 6.3 系统健康监控(依赖主线五每日信号)💤(随 6.2 搁置)

> **注**:下文对照基准整个建在 6.2 的 bootstrap 分布上,随 6.2 判为伪需求一并搁置。核心问题(回撤中「系统坏了还是正常」)仍成立,但设计自承短实盘期「灯几乎恒绿」意义薄;真要做需改「对照真实历史各段回撤」口径,届时重想。原设计保留备查。

### 回答的唯一问题

回撤中:「**是系统失效了,还是历史上就该有的正常回撤?**」——这是杀死执行纪律的头号问题,情绪给的答案永远是「失效了,快改」。

### 设计

- 数据源零新增:每日信号任务全量重放,本就得到模型账户到今日的权益(`SignalRun.equity`,见 daily-signals 设计)→ 上线日起的**实盘期模型净值**序列免费产生。
- 对照系 = 6.2 的 bootstrap 分布:把实盘期长度 L 代入,「历史平行路径跑 L 天的收益/回撤分布」vs 实盘期实际值。
- **三档状态,不给建议**(中性工具,决策归用户):
  - 🟢 正常:实盘期回撤/收益落在历史 80% 区间内;
  - 🟡 偏离观察:落在 80%~95% 尾部;
  - 🔴 超出历史经验:超出 95% 尾部——历史上没见过这么差的 L 天,系统假设可能已变(拥挤/制度变化),值得人工复核。
- 呈现:「今日信号」页每个 live 策略卡带状态灯 + 点开看「实盘净值 vs 期望带」图(期望带 = 分位数扇形)。
- 统计上的诚实:实盘期短时区间极宽、灯几乎恒绿——这是**正确的**(短期说明不了任何事),文案要写明,防用户拿三天实盘判系统生死(那正是要防的行为本身)。

### 验收

用历史数据模拟:把某策略截到 2022 年「上线」,重放 2023-2024 当实盘期,状态灯变化符合直觉(如微盘反转策略在 2024-01 微盘崩盘时应打 🔴)。

## 6.4 策略容量测算

### 为什么(个人视角反用机构概念)

机构问「这策略装得下多少亿」;个人问反面:「**我的 edge 活在多少资金以内**」。资金小是散户唯一的结构性优势(容量受限的 edge 机构进不去),这个数字告诉你优势边界在哪、账户涨到多大要换策略。

### 实现要点

- 复用已有滑点模型(`execPrice` 的冲击项按订单额/当日成交额缩放——机制已在):`initialCash` 网格(如 50万/200万/1000万/5000万/2亿)跑同一策略 → 年化收益 vs 资金规模的衰减曲线。
- 骑在 1.2 参数扫描基础设施上(initialCash 就是参数),专用呈现:衰减曲线 + 标注「滑点年损耗超过 1% / 超过历史年化一半」的资金位。
- 注意:整手取整在小资金端也会造成偏差(50 万买不齐 20 只高价股),曲线两端都有信息。

### 验收

微盘高换手策略呈现明显容量上限(千万级即衰减),大盘低换手策略曲线平坦——分化符合机理。

## 6.5 股息税按持有期(💤 依赖 1.3 A2 现金分红事件化)

- A 股个人:资本利得免税;**红利税按持有期**——持有 >1 年免,1 个月~1 年 10%,<1 个月 20%(卖出时倒扣)。
- 对高分红策略(曾庆辉线)这是真实成本项,且**奖励长持**——与策略气质同向,量化出来能加深「拿得住」的理由。
- 实现:A2 Model B(现金分红事件化)落地后,分红入账时按该笔持仓的持有期计税;成本模型加开关 `dividendTax: 'individual' | 'none'`。在 A2 之前无法做(现在分红藏在复权因子里,没有事件),**明确排在 1.3 之后**。
